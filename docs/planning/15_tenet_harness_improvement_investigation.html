Details When You Have Focus

Evidence: start_job updates DB status

dispatchJob rejects non-pending jobs, then writes status: running, startedAt, and lastHeartbeat. startJob creates a pending ad-hoc job, then immediately writes running.

Relevant source: src/core/job-manager.ts.

Evidence: sequential eval leaves some jobs pending by design

When eval_parallel_safe is false or missing, tenet_start_eval starts code critic but creates test critic and Playwright as pending child jobs with auto_dispatch_on_parent_complete.

Relevant source: src/mcp/tools/tenet-start-eval.ts.

Evidence: old remediation stop wording

The old tenet_request_remediation returned a message ending with End your turn now.. That should be scoped to the report-only worker, not the whole orchestrator.

Relevant source: It has been replaced by src/mcp/tools/tenet-report-blocking-finding.ts.

Evidence: restart recovery can reset running to pending

resetOrphanedJobs changes jobs from running to pending if their server_id does not match the current MCP server. This is recovery behavior, not necessarily a bug.

Relevant source: src/core/state-store.ts.
